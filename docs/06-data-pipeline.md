# 06. 산/등산로 데이터 파이프라인

> 시딩 3단계: **v0 수동(3~5산) → v1 ETL(수도권 20~30산) → v2 전국(100대 명산)**. 스키마는 [02](./02-backend-spec.md), checkpoint 정책은 [03 §1](./03-verification.md).

## 1. v0 — 수동 시딩 (3~5산)

대상: 북한산, 관악산, 청계산 (+아차산, 수락산 후보) — 수도권 다등반 산으로 데모 시나리오(서울 근교)와 일치.

- 방법: 산림청 등산로 SHP에서 해당 산 구간을 QGIS 등으로 추출·병합하거나, 공개 GPX를 참조해 **SQL insert로 직접 작성해도 정당하다** — v0의 목적은 파이프라인이 아니라 데모 루프다. 산당 코스 2~4개.
- `checkpoint_point`: **수동 지정**(정상부 좌표 또는 코스 종점). 자동 산출은 하지 않는다 — 등산로 SHP는 2D(Z값 없음)라 "경로 최고 고도점" 산출이 불가능하고, DEM(수치표고모델) 결합은 스코프가 배로 커진다([v2] 재검토).
- `verify_radius_m`: 기본 150, 정상부가 넓거나 통제된 산은 산별 조정.
- `difficulty`: 원천 난이도값이 있으면 §3 규칙, 없으면 소요시간·경사 기준 수동 판정 + `source_difficulty_raw`에 근거 메모.

## 2. 원천 데이터 (v1~)

| 소스 | 내용 | 포맷 | 역할 |
|---|---|---|---|
| **산림청 등산로 공간정보** (공공데이터 개방목록) | 전국 ~33,000km 구간 폴리라인. 속성: 구간명, 길이, 경사, **난이도값**, 소요시간 | Shapefile | **주 시딩 소스** |
| 산림청 산 정보 조회 API (data.go.kr) | 산명, 소재지, 높이, 개요 | OpenAPI | mountains 메타 보강 |
| 전국등산로표준데이터 (data.go.kr) | 지자체 갱신 노선·난이도 | CSV/OpenAPI | 교차 검증 (커버리지 불균일) |
| 한국등산·트레킹지원센터 | 100대 명산 등 GPX | GPX | 교차 검증 |

- **좌표계: 산림청 등산로는 UTM-K, EPSG:5179 단일.** `ogr2ogr`가 .prj를 읽지만 문서·스크립트에 명시.
- **인코딩: SHP DBF는 EUC-KR/CP949** — 한국 공공 SHP의 고질 함정. `ogr2ogr --config SHAPE_ENCODING CP949` 필수.
- **최신성 한계 명시**: 2017년 정부3.0으로 구축·개방된 데이터로 갱신 주기가 불분명 — 폐쇄·변경 등산로가 반영 안 됐을 수 있다. 실제 등산로와의 불일치는 인증 반경 실패로 직결되므로, 관대 정책([03 §3](./03-verification.md))이 이 리스크의 완충이기도 하다.

### 라이선스 체크리스트 (사용 전 데이터셋별 확인)

- [ ] 공공누리 유형 확인 (제1~4유형 — 상업적 이용·변형 가능 여부)
- [ ] 출처 표시 문구 확정 (앱 내 고지 위치: 설정 > 데이터 출처)
- [ ] 다운로드 방식 확인 (즉시 다운로드 vs 활용신청·승인 대기)
- [ ] vworld/배경지도는 지도 SDK 약관과 별개임을 구분

## 3. v1 — ETL 파이프라인 (`tools/etl/`, 예정)

```
[SHP 다운로드] → ogr2ogr (CP949, EPSG:5179 → 4326)
  → ① 구간(segment) → 코스 조립     ← 파이프라인 최대 난제
  → ② 난이도 정규화
  → ③ checkpoint 산출(종점) 또는 수동 오버라이드
  → ④ PostGIS upsert (source_id 기준)
  → ⑤ [v1] path_low 사전 계산 (EPSG:5179 투영 → ST_SimplifyPreserveTopology 50m → 4326)
```

1. **구간→코스 조립**: 원천은 앱의 "코스"(들머리→체크포인트)가 아니라 구간 단위 폴리라인이다. 기하 연속성(끝점 스냅)으로 후보 체인을 만들고 **최종 확정은 반수동 큐레이션** — v1 규모(20~30산)에서 정당한 선택이며, 이 결정 자체를 기록한다. 완전 자동 조립은 v2 과제.
2. **난이도 정규화**: 구간 난이도의 **최댓값 = 코스 난이도** ("가장 어려운 구간이 코스 난이도"). 원본은 `source_difficulty_raw`에 보존.
3. checkpoint: 조립된 코스의 종점. 정상 경유 코스는 정상 좌표로 수동 스냅.
4. upsert: `courses.source_id`(원천 구간 ID 조합) 기준 — 재시딩 시 climbs FK 보존([02 §3](./02-backend-spec.md)).
5. 지역 파라미터화: 스크립트를 시도/산 코드 인자로 실행 — v2 전국 확장은 스크립트 재실행으로.

## 4. 갱신 시나리오

- 원천 데이터 갱신 → 동일 `source_id` upsert → 지오메트리·속성만 교체, climb 기록 무손상.
- 코스 폐지 → soft delete(`deleted_at`) — 기존 climb의 색칠은 유지하되 신규 인증 차단.
