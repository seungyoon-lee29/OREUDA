# 05. 디자인 결정 기록 (Design Decisions)

> 토큰 시스템이 아니라 **결정 기록**이다 — 값은 필요한 만큼만. 상태의 전이 조건은 [04](./04-client-architecture.md) 정본을 따르고, 이 문서는 각 상태가 **어떻게 보이고 무엇을 말하는지**를 담당한다. §4 매트릭스와 §5 카피 표는 [04 §4.3](./04-client-architecture.md) 매핑 표의 클라 표시 상태를 1:1 커버한다.

## 1. 디자인 원칙 3가지

1. **야외 우선** — 직사광·장갑·추위·배터리 불안이 디폴트 컨텍스트다.
2. **원탭 인증** — 콜드스타트→인증 완료까지 **필수 탭 ≤2회**(앱 아이콘 제외: 인증 CTA 탭 → 코스 선택 탭). 인증 플로우에 확인 다이얼로그·스와이프·롱프레스 금지. **예외**: 판정 marginal(반경 밖/저정확도)은 소프트 확인 1탭('그래도 인증하기')이 추가된다 — 막다른 거절 대신 확인 후 수용이 관대 정책([03 §3](./03-verification.md)) 정합.
3. **색칠이 보상** — 이 앱의 리워드는 포인트·레벨이 아니라 지도가 물드는 순간이다. 모든 연출 투자는 여기에 집중한다.

## 2. 사용 컨텍스트 → 디자인 요구사항

| 제약 | 요구사항 |
|---|---|
| 직사광 | 라이트 + 고대비 (본문 대비 7:1 목표, #767676보다 밝은 회색 텍스트 금지, 폰트 웨이트 ≥500). **다크맵 불가** — 네이버 SDK 야간 모드는 Navi 지도 유형 전용([ADR-001](./adr/ADR-001-map-sdk.md)) |
| 장갑 | 인증 CTA 높이 **64dp**(폭 = 화면−32dp), 기타 야외 사용 버튼 **≥56dp** (플랫폼 최소 44/48dp는 장갑에 부족) |
| 추위/조급함 | 원칙 2의 탭 예산. 사진 스텝 없음(v0), [v1] 도입 시에도 캡처 **완료 후** 스킵 가능 스텝 |
| 배터리 불안 | GPS 고정밀은 인증 순간만([04 §5](./04-client-architecture.md)). 캡처는 3초 내 완료 + 명시적 완료 피드백(햅틱+풀스크린) — "된 건가?" 재시도 방지 |
| 오프라인(정상=무신호) | 위저드 지도 비의존, 시퀀스 타일 폴백(§6), "전송 대기" 노출은 기록 탭에 숨기지 않고 홈 지도에도 뱃지 |
| 상태 표현 | 색 단독 인코딩 금지 — 색+아이콘+텍스트 3중 |

## 3. 컬러 시스템

### 3.1 난이도 3색 — Okabe-Ito 기반 (색약 안전)

| 난이도 | hex | 선정 근거 |
|---|---|---|
| 쉬움 | `#009E73` (bluish green) | Okabe-Ito 팔레트 — 세 색이 hue와 명도가 동시에 갈라져 적록색약 시뮬레이션에서도 구분됨 (한국 남성 색각이상 약 5~6%, 주 타깃 성비 고려 시 무시 불가) |
| 보통 | `#0072B2` (blue) | |
| 어려움 | `#D55E00` (vermillion) | |

- 문서/포트폴리오에 색약 시뮬레이션 캡처 1장 포함 권장.
- 보조 인코딩: 코스 리스트에 "상/중/하" 텍스트 뱃지 (원칙: 색 단독 금지).
- 시맨틱 색: 성공 `#009E73` 재사용 / 대기 `#8A8A8E`+시계 아이콘 / 오류 `#C43E00`.

### 3.2 코스 선 스타일 매트릭스

> 행 이름 = [04 §4.3](./04-client-architecture.md) 클라 표시 상태. 전이 조건은 04가 정본.

| 상태 | 색 | 불투명도 | 굵기 | 케이싱(외곽선) | 뱃지 |
|---|---|---|---|---|---|
| 미완등 | 난이도색 | 55% | 2.5dp | 없음 | 없음 |
| **pending** | 난이도색 원색 | 100% | 4dp | 흰색 1.5dp | 종점에 시계 뱃지 |
| **verified** | 난이도색 원색 | 100% | 5dp | 흰색 1.5dp | 없음 |
| verified+flags | **verified와 완전 동일** | — | — | — | **구분 뱃지 없음** — 관대 정책의 취지([03 §3](./03-verification.md)). flags는 v0 UI 어디에도 미노출 |
| failed (4xx) | 미완등으로 롤백 | — | — | — | 기록 탭 고지만 |
| expired | — | — | — | — | **[v1]** — v0에 만료 없음 |

- **미완등도 난이도 색을 유지한다**(저채도가 아니라 저불투명+얇은 선). 난이도 정보가 정작 필요한 순간은 미완등 코스를 고를 때다.
- 흰색 케이싱이 배경 분리의 핵심 — 배경이 무엇이든 대비를 보장한다.

## 4. 지도 스타일 결정

| 화면 | 지도 유형 | 근거 |
|---|---|---|
| 홈 정복 지도 | **Basic** + `lightness` −0.1~−0.2 톤다운 | 저채도 배경이라 색칠이 가장 잘 뜸, 톤다운으로 원색 대비 추가 확보 |
| 산 상세 / 인증 맥락 | **Terrain** | 음영기복으로 산 형태 인지 — 등산 앱 정체성 |
| 위성 | **배제** | 숲의 짙은 녹색과 코스선 충돌, 직사광 저대비, 데이터량 |

- 다크맵: 불가(§2). 산악 레이어 그룹(SDK 자체 등산로 선): 우리 폴리라인과 시각 충돌 — **홈에서 off**, 산 상세에서 on/off는 프로토타입에서 판정([ADR-001](./adr/ADR-001-map-sdk.md) 결정 항목).

## 5. 상태별 화면·카피 표 (캡처 위저드)

> 상태 키 = [04 §4.1](./04-client-architecture.md). 이 표가 카피의 정본이다(04에는 카피 없음).

| 상태 | 화면 | 카피 |
|---|---|---|
| requesting_permission | 프라이밍 1장 → 시스템 다이얼로그 | "정상에서 위치로 완등을 인증해요" |
| permission_denied | 설정 딥링크 버튼 | "위치 권한이 있어야 인증할 수 있어요" |
| acquiring_fix | 로딩 + 위성 아이콘 | "위치를 확인하는 중…" |
| fix_failed | 재시도 버튼(64dp) | "위치를 찾지 못했어요. 다시 시도해 주세요" |
| confirm_marginal | 사유 전부 표시 + '그래도 인증하기'(64dp) + 재시도 | 거리: "체크포인트에서 조금 떨어져 있어요"(1km 초과면 "멀리") / 정확도: "GPS 정확도가 낮아요" · "GPS 오차 ±140m" — 복합이면 병기. 공통 꼬리: "정상에 있다면 그대로 인증하세요. 완등으로 기록돼요" |
| captured | 성공 시퀀스(§6) | — |
| (큐) queued | 홈 뱃지 + 기록 탭 행 | "전송 대기 1건 — 연결되면 자동으로 제출돼요. 늦어도 다음에 앱을 열 때" |
| (큐) confirmed→verified | 무연출 | 뱃지 조용히 제거 |
| (큐) rejected(duplicate) | **지도 무연출** | "이미 인증된 코스예요. 기존 기록이 유지돼요" |
| (큐) failed(4xx) | 기록 탭 고지 | "제출에 문제가 있어 저장하지 못했어요" |

## 6. 핵심 모먼트 — 인증 성공 시퀀스

> 트리거·분기 조건은 [04 §5](./04-client-architecture.md)(연출 소스 = outbox insert 이벤트). 여기는 연출 스펙.

**대원칙: 연출은 하나, 착지만 둘.** 축하 시퀀스는 온라인/오프라인 완전 동일 — 오프라인 완등이 열등한 승리로 느껴지면 안 된다. 갈라지는 것은 마지막 정착 프레임뿐.

### 최소안 (v0 커밋)

| t | 연출 |
|---|---|
| 0ms | 성공 햅틱(notification-success) + 풀스크린 체크 |
| 600ms | 지도 카메라가 코스 boundingBox로 fit (ease-out) |
| 1.2s | 코스 선 원색 페이드인 + 글로우 펄스 2회 |
| 2.2s | **"나의 12번째 산"** 카운터 카운트업 |
| 착지 | CTA **"기록 보기"** (S5로 브리지) — [v1] 여기에 공유 CTA·사진 스텝 삽입 지점 예약 |

- **착지 이원화**: 온라인(동기 verified) → 선이 곧장 verified 스타일(5dp+케이싱), 뱃지 없음 / 오프라인·타임아웃 → 동일 시퀀스 후 정착 시 시계 뱃지 + 카피 1줄("서버 확인 대기 — 늦어도 다음 앱 열 때").
- **동기 응답 게이팅 상한 2~3초**: 온라인이면 시퀀스 시작을 서버 응답에 최대 2~3초만 게이팅 — 중복(rejected)이 시퀀스 **전에** 걸러져 "축하 후 취소"가 원천 차단되고, flagged는 받아도 그대로 재생(색칠 유지). 타임아웃이면 낙관 경로 폴백 — 축하를 네트워크에 인질 잡히지 않는다.
- **타일 미로드 폴백(오프라인 정상)**: 지도 연출(카메라 fit·선 페이드인) 생략 → 풀스크린 체크+햅틱+**카운터+코스명 카드** 중심. 지도 색칠은 온라인 복귀 후 자연 반영.

### 목표안 [v1]

폴리라인 N등분 세그먼트 순차 추가로 시작점→체크포인트 "그려지는" 연출 (SDK에 progressive-draw API가 없어 세그먼트 방식 — 렌더 비용은 04 §7 제약 하에서 검증).

## 7. 저줌 정복 마커

줌아웃 시 코스선은 서브픽셀 얼룩이라 무의미 — 히스테리시스([04 §7](./04-client-architecture.md)) 아래에서는 산 마커만 남는다. **마커가 정복 상태를 표현해야 전국 뷰에서도 "정복 지도" 서사가 유지된다**:

- 미정복: 아웃라인 마커 (채움 없음)
- 정복(해당 산에 verified/pending 코스 ≥1): 난이도색 채움 + 체크
- 토글 구조: 코스선이 마커를 **대체하는 게 아니라 추가**되는 구조 (고줌 = 마커+코스선).

## 8. 카운터 — 진행률 링 폐지 (v0)

v0 시딩이 3~5산이므로 "n/5 링"은 첫 완등에 20%가 차는 우스운 UI가 된다. **v0은 무한 카운터 "나의 n번째 산"**(소스: `GET /v1/me/climbs`의 `totalMountains`) — 빈 상태·시퀀스·기록 탭에서 동일 컴포넌트. 진행률 링+분모는 100대 명산 프리셋(로드맵)과 함께 부활한다.

## 9. 빈 상태 "도화지"와 온보딩

- **첫 지도 = 도화지**: 미완등 저불투명 선이 깔린 지도는 그 자체가 "칠해야 할 칸"으로 읽힌다. + "가장 가까운 산: 관악산 2.3km" 추천 카드 1장.
  - 추천 폴백: 위치 권한 전/시딩 산에서 원거리 → 시딩 산 고정 리스트("이런 산부터 칠해보세요").
- **권한 온보딩**: 첫 실행 즉시 시스템 다이얼로그 금지. 지도를 권한 없이 서울 디폴트 중심으로 먼저 렌더 → "내 주변 산 보기" 탭 또는 첫 인증 시도 시점에 가치 프라이밍 1장 → 시스템 다이얼로그. **지도·완등 인증은 포그라운드 위치만 요청**. 등반 위젯의 세션 한정 백그라운드 위치는 온보딩이 아니라 **등반 시작 시** 별도로 요청하고, iOS '항상 허용'은 사용자가 원할 때만 업그레이드(강제 아님) — [07 §2](./07-security-privacy.md) 동의 UX와 연결.
- 가입 게이트: 선가입([01 §7](./01-product-spec.md)) — 가입 화면도 §2의 대비·타겟 규칙을 따르되 야외 최적화 대상은 아니다.

## 10. v0 기록 리스트 아이템 (사진 없음)

사진이 v1로 빠지면서 리스트가 텍스트만 남는 문제 → 썸네일 슬롯에 **난이도색 코스 선 미니 렌더**(Skia 불필요 — 프리페치된 path를 react-native-svg 또는 단순 Polyline 축소 렌더, 실측 후 캐시). 폴백: 난이도색 원형 + 산 이니셜. 구성: [코스 미니 렌더] 산 이름 · 코스명 · 날짜 · 난이도 뱃지(텍스트 병기).

## 11. 공유 카드 [v1] — "완등 카드"

전국 색칠 지도 카드는 몇 달 쓴 뒤에야 자랑거리(시즌 결산용 — [v2]). v1 카드는 **"오늘, 이 산, 나"**:

- 구성: 사용자 사진 배경(없으면 지형 음영 그라디언트+대형 타이포 폴백) + 해당 코스 색칠 선 오버레이(Skia, [04 §8](./04-client-architecture.md)) + 산 이름·고도·날짜 + **"나의 12번째 산"** 대형 카운트 + 로고 워터마크.
- 포맷: 4:5(1080×1350, 인스타 피드) 기본 + 9:16 스토리 변형.

## 12. 비범위 (v0~v1)

다크 모드, 태블릿 레이아웃, 커스텀 지도 스타일, 시퀀스 목표안 이상의 애니메이션, 접근성 스크린리더 최적화(색약 대응은 §3에서 커버), 로티/일러스트 리소스.
