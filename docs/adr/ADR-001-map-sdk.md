# ADR-001. 지도 SDK — @mj-studio/react-native-naver-map

- 상태: 승인 (2026-07)
- 결정자: 팀 리뷰 3라운드 (기술 검증 → 적대적 리뷰 → 교차 피드백)

## 상황

국내 등산 앱. 코스 폴리라인 다수 + 마커 + 위치 오버레이를 지도 위에 그려야 하고, 한국 산지의 지형 표현 품질이 UX를 좌우한다. RN(Expo dev client) 환경.

## 결정

**`@mj-studio/react-native-naver-map` v2.x** (네이버 지도 네이티브 SDK 래퍼, NCP 신규 "Maps" 상품).

## 근거 (2026-07 검증)

- Expo config plugin 공식 지원 (CNG/dev client — **Expo Go 불가**), New Architecture 전용(v2.x), iOS/Android × 신구 아키텍처 조합 테스트 명시. 유지보수 활발(2026-06 기준 최근 릴리스).
- Polyline / Path / Polygon / LocationOverlay + **마커 클러스터링 내장**(`clusters` prop — 자체 구현 금지, [04 §7](../04-client-architecture.md)).
- 요금: NCP **신규 Maps 상품** 기준 Mobile Dynamic Map **월 1억 건 무료**(맵 뷰 onCreate 1회=1건) — 포트폴리오~소규모 서비스에서 사실상 무료. 구 "AI NAVER API" 지도 상품은 신규 신청 차단됨 — **반드시 신규 Maps 상품으로 가입**.

## 기각한 대안

| 대안 | 기각 사유 |
|---|---|
| 카카오맵 | **공식 RN SDK 없음** — WebView 우회는 폴리라인 다수+위치 오버레이에 부적합 |
| react-native-maps (Google) | 국내 지도 상세도·등고선 표현 열세 (지도데이터 반출 제한) |
| MapLibre + vworld 타일 | 유효한 **백업 옵션** — 타일 스타일·캐싱·사용량 관리를 직접 해야 하고 등산 지형 스타일 커스텀 부담. 아래 재검토 트리거 참조 |

## 수용한 제약과 리스크

1. **오프라인 지도 미지원** — 네이버 SDK는 오프라인 타일을 지원하지 않는다(내부 캐시는 제어 불가). 앱 레벨 대응으로 흡수: 위저드 지도 비의존 + 코스 데이터 프리페치 + 타일 미로드 연출 폴백 ([04 §5](../04-client-architecture.md), [05 §6](../05-design.md)).
2. **야간 모드는 Navi 지도 유형 전용** — Basic/Terrain에서 다크맵 불가 → 라이트+고대비 확정 ([05 §2](../05-design.md)).
3. **산악 레이어 그룹**(SDK 자체 등산로 선)이 우리 폴리라인과 시각 충돌 — 홈 off, 산 상세 on/off는 프로토타입 판정 ([05 §4](../05-design.md)).
4. **스냅샷/캡처 API 없음** — 공유 카드는 Skia 직접 렌더로 우회 ([04 §8](../04-client-architecture.md)).
5. **NCP 가입**: 개인 가능하나 **결제수단(카드) 등록 필수**, 전화번호 기준 1계정만 무료 이용량 대상(대표계정) — 비대표 계정 호출은 즉시 과금. 쿼터 알림·한도 설정을 가입 직후 구성할 것.
6. Expo SDK 버전 고정: 라이브러리 릴리스 노트의 지원 매트릭스 확인 후 pin.

## 재검토 트리거

**오프라인 지도 타일이 하드 요구사항이 되는 순간**(사용자 피드백으로 "산에서 지도가 안 보인다"가 상위 불만이 될 때) MapLibre + vworld(오프라인 팩 지원)로 전환을 재평가한다.
