# 07. 보안·프라이버시·심사 체크리스트

> 1~2페이지 체크리스트로 유지한다. 버전 마킹은 [01 §5](./01-product-spec.md) 컷 표 기준 — **v0 항목만이 데모 전 필수**다.

## 1. 데이터 보호 (v0)

- [ ] 비밀번호: bcrypt/argon2 해시 (v0)
- [ ] 토큰: access 단수명 + 고정 refresh(SecureStore 보관). rotation은 [v1]
- [ ] API: 전 구간 HTTPS, auth throttle 5회/분 ([02 §5](./02-backend-spec.md)) (v0)
- [ ] 위치 데이터 최소 수집: 인증 시점 1포인트만 저장(`verified_point`) — 이동 경로·상시 위치는 수집하지 않음 (v0 설계 자체가 최소화)
- [ ] Sentry에 위치 좌표 미전송 (breadcrumb에서 좌표 필드 스크럽) (v0)

## 2. 위치정보법 (한국)

- 위치정보 = 국내법상 별도 규율 대상. **개인위치정보를 수집·이용·제공하는 서비스는 위치기반서비스사업자 신고 대상**(방통위, 소상공인·1인 개발자는 간이신고 제도 확인).
- [ ] [v2 게이트 — 스토어 공개 전] 위치기반서비스사업자 신고(간이신고 요건 확인)
- [ ] [v2] 위치정보 이용약관 별도 게시 + 개인위치정보 수집·이용 동의 절차
- [ ] (v0부터 준수) 동의 UX: 위치 권한 요청 전 가치 프라이밍([05 §9](./05-design.md)) — **포그라운드 위치만** 요청, "항상 허용" 불요. 백그라운드 위치는 파티 기능(로드맵)에서만 재검토
- 참고: v0~v1은 비공개 데모/TestFlight 내부 테스트 범위로 운영 — 신고 의무 발생 시점을 스토어 공개와 함께 판단

## 3. 사진 파이프라인 [v1]

- [ ] **EXIF GPS 스트립**: 서버 저장 전 EXIF 위치 메타 제거 (인증샷의 GPS EXIF도 개인위치정보)
- [ ] 비공개 버킷 + 조회도 presigned GET (공개 버킷 금지)
- [ ] 업로드: 서버 발급 key(`pending/{userId}/{uuid}`)만 수락, presigned POST에 content-length-range/type 조건, lifecycle rule로 pending/ 48h 자동 삭제
- [ ] 리사이징/썸네일 파이프라인 결정 (v1 설계 시)

## 4. 스토어 심사 요건

- [ ] [v2] **Apple 5.1.1(v) 계정 삭제**: 정책 문서가 아니라 **앱 내 삭제 버튼 + 계정 삭제 API** 필수
- [ ] [v2] 양 스토어 개인정보처리방침 URL
- [ ] [v0부터] iOS 위치 권한 문구(`NSLocationWhenInUseUsageDescription`)를 심사 관점으로 작성 — "완등 인증에 현재 위치를 사용합니다"
- [ ] [v1] 카카오 로그인 도입 시 → **Sign in with Apple 의무 발동** (제3자 로그인 제공 시) — v2 Apple 로그인 일정과 연계
- [ ] [로드맵] 파티 위치공유 = Android 백그라운드 위치 권한 선언 + iOS Always 정당화 — 심사 난이도 최상, 별도 설계 문서 필요

## 5. 탈퇴·파기 (v0 정책, v2 완전 구현)

- soft delete(`deleted_at`)와 파기 의무의 긴장: 탈퇴 시 개인정보(이메일, 해시)는 즉시 익명화, 통계성 기록은 비식별 보존 여부를 약관에 명시
- [ ] [v2] 탈퇴 API + 사진 객체 삭제 + 30일 유예 정책 결정

## 6. 알려진 수용 리스크 (위협모델 요약 — 정본: [03 §5](./03-verification.md))

- v0에 기기 무결성 증명 없음(curl 제출 가능) — 부정 인센티브 낮음을 근거로 수용, [v2] Play Integrity/App Attest
- iOS mock location 탐지 공백 — 명시적 수용
- `captured_at` 클라 주장값 — sanity(미래 금지·≤submitted_at)만, 만료 없음(v0)
